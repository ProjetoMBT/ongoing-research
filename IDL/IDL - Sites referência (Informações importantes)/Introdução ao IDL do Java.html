Introdução ao Java ™ IDL

Java ™ IDL é uma tecnologia para objetos distribuídos - ou seja, objetos interagindo em diferentes plataformas em uma rede. O Java IDL permite que os objetos interajam independentemente de estarem escritos na linguagem de programação Java ou em outra linguagem como C, C ++, COBOL ou outros.

Isso é possível porque o IDL do Java é baseado no Common Object Request Brokerage Architecture (CORBA), um modelo de objeto distribuído padrão do setor. Uma característica fundamental do CORBA é o IDL, uma linguagem de linguagem neutra. Cada linguagem que suporta CORBA tem seu próprio mapeamento IDL - e como seu nome indica, o Java IDL suporta o mapeamento para Java. Para saber mais sobre o mapeamento de linguagem IDL para Java, consulte mapeamento de linguagem IDL-para-Java .

Para suportar a interação entre objetos em programas separados, o Java IDL fornece um Object Request Broker ou ORB. O ORB é uma biblioteca de classes que permite comunicação de baixo nível entre aplicativos Java IDL e outros aplicativos compatíveis com CORBA.

Este tutorial ensina as tarefas básicas necessárias para construir um aplicativo distribuído CORBA usando Java IDL. Você irá construir o clássico programa "Hello World" como um aplicativo distribuído. O programa Hello World tem uma única operação que retorna uma seqüência de caracteres a ser impresso.

Qualquer relacionamento entre objetos distribuídos tem dois lados: o cliente e o servidor. O servidor fornece uma interface remota e o cliente chama uma interface remota. Essas relações são comuns à maioria dos padrões de objetos distribuídos, incluindo Java Remote Method Invocation ( RMI , RMI-IIOP ) e CORBA . Observe que, neste contexto, os termos cliente e servidor definem nível de objeto ao invés de interação em nível de aplicativo - qualquer aplicativo pode ser um servidor para alguns objetos e um cliente de terceiros. Na verdade, um único objeto poderia ser o cliente de uma interface fornecida por um objeto remoto e, ao mesmo tempo, implementar uma interface a ser chamada remotamente por outros objetos.

Esta figura mostra como um objeto distribuído de um método é compartilhado entre um cliente e servidor CORBA para implementar o aplicativo clássico "Hello World".

Um objeto distribuído de um método compartilhado entre um cliente e servidor CORBA.

No lado do cliente, o aplicativo inclui uma referência para o objeto remoto. A referência de objeto tem um método stub, que é um stand-in para o método sendo chamado remotamente. O stub é realmente conectado no ORB, de modo que chamá-lo invoca capacidades de conexão do ORB, que encaminha a invocação para o servidor.

No lado do servidor, o ORB usa o código de esqueleto para traduzir a chamada remota em uma chamada de método no objeto local. O esqueleto traduz a chamada e quaisquer parâmetros para seu formato específico de implementação e chama o método que está sendo invocado. Quando o método retorna, o código de esqueleto transforma resultados ou erros e os envia de volta para o cliente através dos ORBs.

Entre os ORBs, a comunicação prossegue por meio de um protocolo compartilhado, IIOP - o Internet Inter-ORB Protocol. O IIOP, que é baseado no protocolo de Internet TCP / IP padrão, define como os ORBs compatíveis com CORBA passam as informações de um lado para o outro. Como CORBA e IDL, o padrão IIOP é definido pelo OMG, o Object Management Group.

O Processo de Desenvolvimento IDL Java eo Tutorial Hello World

Este tutorial ensina as tarefas básicas na construção de um aplicativo distribuído CORBA usando Java IDL. Você irá construir o clássico programa "Hello World" como um aplicativo distribuído. O programa "Hello World" tem uma única operação que retorna uma seqüência de caracteres a ser impressa.

Apesar do seu design simples, o programa Hello World permite que você aprenda e experimente todas as tarefas necessárias para desenvolver quase qualquer programa CORBA que use a invocação estática. As etapas a seguir fornecem um guia geral para projetar e desenvolver um aplicativo de objeto distribuído com Java IDL. Links para as etapas relevantes do tutorial irão guiá-lo através da criação deste aplicativo de exemplo.

  1. Definir a interface remota

    Você define a interface para o objeto remoto usando o OMG Interface Definition Langauge (IDL). Você usa IDL em vez da idljlinguagem Java porque o compilador mapeia automaticamente a partir de IDL, gerando todos os arquivos de código-fonte e esqueleto da linguagem Java, juntamente com o código de infraestrutura para conexão com o ORB. Além disso, usando IDL, você torna possível para desenvolvedores para implementar clientes e servidores em qualquer outro linguagem compatível com CORBA.

    Observe que se você estiver implementando um cliente para um serviço CORBA existente ou um servidor para um cliente existente, você obteria as interfaces IDL do implementador - como um provedor de serviços ou fornecedor. Em seguida, execute o idlj compilador sobre essas interfaces e siga estas etapas.

    Escrever o arquivo IDL neste tutorial orienta você através da definição da interface remota para o exemplo simples "Hello World".

  2. Compilar a interface remota

    Quando você executa o idljcompilador sobre seu arquivo de definição de interface, ele gera a versão Java da interface, bem como os arquivos de código de classe para os stubs e esqueletos que permitem que seus aplicativos se liguem ao ORB.

    Mapeando Hello.idl para Java neste tutorial orienta você através destas etapas para o exemplo simples "Hello World".

  3. Implementar o servidor

    Depois de executar o idljcompilador, você pode usar os esqueletos que ele gera para montar seu aplicativo de servidor. Além de implementar os métodos da interface remota, o código do servidor inclui um mecanismo para iniciar o ORB e aguardar a invocação de um cliente remoto.

    O desenvolvimento do servidor Hello World orienta você através da criação de um servidor simples para o aplicativo "Hello World".

  4. Implementar o cliente

    Da mesma forma, você usar os stubs gerados pelo compilador idljcomo a base do seu aplicativo cliente. O código do cliente baseia-se nos stubs para iniciar seu ORB, procure o servidor usando o serviço de nome fornecido com IDL Java, obtenha uma referência para o objeto remoto e chame seu método.

    O desenvolvimento de um aplicativo cliente orienta você através da criação de um aplicativo cliente simples.

  5. Iniciar as aplicações

    Depois de implementar um servidor e um cliente, você pode iniciar o serviço de nome e, em seguida, iniciar o servidor e, em seguida, executar o cliente.

    Executando o aplicativo Hello World orienta você através da execução do servidor e do programa cliente que, juntos, compõem o aplicativo "Hello World" eo serviço de nome que permite que eles encontrem um ao outro.

O uso de Referências de Objeto Referenciadas orienta você ao fazer uma referência de objeto quando não há um serviço de nomeação.

Executar o aplicativo Hello World em duas máquinas descreve uma maneira de distribuir o aplicativo simples em duas máquinas - um cliente e um servidor.
