Java IDL: IDL para mapeamento de linguagem Java

Esta versão do Java ™ 2 Platform Standard Edition 5.0 inclui idlj , um compilador que lê um arquivo IDL (Interface Definition Language) do Grupo de Gerenciamento de Objetos (OMG) e o traduz ou o mapeia para uma interface Java. Dependendo das opções de linha de comando usadas, o compilador idlj cria stub, skeleton, helper, holder e outros arquivos conforme necessário. Estes arquivos .java são gerados a partir do arquivo IDL de acordo com o mapeamento especificado no documento OMG (formato pdf) OMG IDL para Java Language Mapping Specification, ptc , 00-01-08 , com as revisões indicadas em ptc / 00-11- 03, O novo IDL para Mapeamento Java Capítulo . A versão 00-01-08 do mapeamento IDL para linguagem Java está alinhada com CORBA versão 2.3.2.

  • A tecnologia CORBA fornecida no J2SE 5.0 está em conformidade com os mapeamentos especificados pelo documento a seguir: Especificações oficiais para suporte CORBA no J2SE 5.0 .

Alguns tópicos da especificação OMG IDL para Java Language Mapping que não são facilmente resumidos são:

  • Seção 1.13, Mapeamento para Tipo de Valor
  • Seção 1.14, Tipos de Caixa de Valor
  • Seção 1.20, Mapeamento do lado do servidor
  • Seção 1.21, Interfaces de portabilidade Java ORB

Visão geral do mapeamento IDL para linguagem Java

Objetos CORBA são definidos em OMG IDL (Object Management Group Interface Definition Language). A sintaxe e a semântica do OMG IDL são definidas no Capítulo 3 da Especificação CORBA / IIOP 2.3.1 . O compilador IDL para Java, idlj , mapeia as interfaces definidas pelo IDL para classes e interfaces Java, conforme definido no OMG IDL para Java Language Mapping Specification , ptc, 00-01-08 e como suportado nesta versão da plataforma Java , Conforme descrito nas Especificações Oficiais para o suporte CORBA no J2SE 5.0 .

Esta visão geral mostra a correspondência entre tipos OML IDL e tipos Java. Observe que OMG IDL, como seu nome indica, define interfaces. Como interfaces Java, interfaces IDL não contêm implementações para suas operações (métodos em Java). Em outras palavras, as interfaces IDL definem apenas a assinatura de uma operação (o nome da operação, o tipo de dados do seu valor de retorno, os tipos de dados dos parâmetros que ele toma e quaisquer exceções que ele gera). As implementações para essas operações precisam ser fornecidas em classes Java escritas por um programador Java.
